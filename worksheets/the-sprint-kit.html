02

Deployment ≠ Release

ship dark · release is a business event
sheet 2 / 5

Deploying code and releasing it to users are two separate events. A feature flag is the seam between them.

tick the rungs you already stand on · climb upward
R1We can deploy without releasing — the seam exists.
R2New work ships behind a flag, off by default.
R3A PO flips the flag in prod (staff-only first) to try it.
R4Release is a business call, decoupled from deploy cadence.
▶ our next rung this sprint: R   owner
// decoder
deploy
put code into the production environment (it may still be off).
release
turn a feature on for users — a separate, deliberate decision.
feature flag
a runtime on/off switch wrapping a piece of code.
staff flag
a flag on only for staff, so they can try it in prod first.

03

Delivered ≠ Done

engineering delivers · the business accepts
sheet 3 / 5

"Delivered" = code live in prod (dark). "Done" = the PO accepted it there. Mind the handoff — and kill the flag after.

tick the rungs you already stand on · climb upward
R1“Deployed” and “accepted” are different words on our board.
R2We gather feedback while a feature is still dark (staff flag, PO in prod).
R3The PO accepts the live feature, and that flips the flag on.
R4After release we consolidate: remove the flag, delete the old path.
▶ our next rung this sprint: R   owner
// decoder
product owner (PO)
the person who decides a feature is good to release.
consolidate
remove the flag + delete the replaced old path — one clean path.
flag debt
flags left in the code long after they should've been removed.

The worktree clean-room

interrupt without disturbing your work
sheet 4 / 5

Handle an interruption in a throwaway workspace, without stashing or disturbing the work you were already doing.

tick the rungs you already stand on · climb upward
R1You can create a git worktree.
R2A hotfix spawns a clean tree off trunk without touching your WIP (wtfix).
R3Keep it rebased, push to trunk, delete when done (wtsync/wtback).
R4Agents / parallel work run in their own trees (claude --worktree).

// decoder
worktree
a second checkout of the same repo in its own folder.
WIP

rebase

wtfix / wtback
shell helpers: spawn a clean tree / clean it up after.
